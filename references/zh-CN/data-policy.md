# 数据政策

> [../data-policy.md](../data-policy.md) 是英文主版本；本文件说明中文用户最关心的记录边界。

jinhua 只记录脱敏后的方法论信号。它不是用户记忆、项目记忆，也不是聊天记录工具。

## 可以记录

- 脱敏后的方法论摘要。
- 抽象出来的方法动作。
- 触发条件。
- 迁移条件。
- 反例或禁用场景。
- 验证路径。
- 方法类型（operator id）。
- 信号强度（strength）和可选信心分（confidence）。
- 用户确认结果。
- 项目身份哈希。
- 压缩后的全局晋升证据。
- 哈希前的可选显式项目身份；这个值只用于生成哈希，不应明文进入全局记录。

## 不能记录

- 用户身份信息。
- 联系方式、账户、密钥、token 或其他凭证。
- 完整用户原文。
- 完整对话。
- 客户名称、公司秘密或敏感项目标识。
- 私人偏好。
- 普通 bug 修复事实。
- 全局晋升记录里的原始项目路径。
- 全局晋升记录里的明文显式项目 id。

## 记录规则

无法安全脱敏时，不要记录。

信号很弱时，直接忽略，不要为了“有记录”写入一条低质量拒绝记录。

高质量信号通常包含：

- 可以复用的方法。
- 清楚的触发条件。
- 合理的迁移条件。
- 已知反例或风险。
- 可执行的验证路径。

## 本地与全局

项目本地 `.jinhua/data/` 可以保存更完整的信号卡。

全局 `global-data/` 只能保存压缩后的晋升证据：

- 方法指纹（method fingerprint）。
- 方法键（method key）。
- 脱敏摘要（summary）。
- 信号卡字段（signal-card fields）。
- 方法类型（operator）。
- 来源类型（source type）。
- 信号强度（strength）。
- 可选信心分（confidence）。
- 项目身份哈希。

不要把本地原始证据复制到全局状态。

使用 `--project-id` 或 `JINHUA_PROJECT_ID` 时，先哈希显式身份，只保存哈希和身份来源。

## 写入边界

自动积累不是后台任务。只有 Skill 被调用，并且模型显式调用 CLI 时，才会写入文件。

如果文件无法写入，要说明持久化失败，并给出本来会写入的脱敏记录。
